# Site de livraison pour un Foodtruck

## A quel besoin répond ce site 

Pour le client:
- Consultation du menu et commander.
- Suivre l'état de ces commandes.
- Avoir un historique (date, prosuits, prix)

Pour le restaurateur :
-Ajouter, modifier et supprimer simplement ces produits.
- Recevoir ces commandes avec une interface et sans erreurs.
- Modifier le statut de la commande pour tenir inforer le client.
- Voir le nombre de commandes réalisées aujourd'hui et son chiffre d'affaires.

## Controle automatique 

- Impossibilité de passer des commandes avec un chiffre d'article négatifs ou un nombre d'article supérieur a 20 pour éviter les manipulation utilisateurs.

## Gestion des accès 

- Page de login forcée pour passer une commandes.
- Les menus commandes et historique ne saffichent pas si l'utilisateur n'est pas connecté.
- Le menu admin et le changement des produits ne s'affiche pas si l'utilisateur n'a pas le rôle administrateur.
